# Стратегия сайзинга медиа в чате

Как чат считает размеры фото, видео, кружков и альбомов. Цель — **как в Telegram**:
картинка сразу встаёт в свой прямоугольник, лента не прыгает, широкие фото тянутся в
ширину, высокие ограничены по высоте, и размер не меняется между лоадером и итогом.

Код: `frontend/src/views/ChatView.vue` (функции `convWidth`, `fitBox`, `photoCaps`,
`videoCaps`, `photoBox`, `videoBoxStyle`, `albumWidth`). Пропорции — `imgAspects` /
`imageAspect` / встроенные в маркер размеры.

## Принципы

1. **Точный резерв места ДО загрузки.** Бокс получает явные `width`/`height` в px из
   пропорций, а не из intrinsic-размера пустого `<img>/<video>` (иначе схлопывается в
   точку и потом прыгает). Пропорции берём по приоритету:
   - для видео — из маркера `@[video](url|poster|bytes|ШxВ)` (embed WxH);
   - из серверного `/api/uploads-dims` (`imageAspect`, переживает очистку кэша);
   - из локального замера при отправке (`imgAspects`, праймится сразу после аплоада);
   - иначе дефолт `1.4` (лёгкий landscape).
2. **Landscape тянется в ширину, portrait ограничен высотой.** Вписываем в
   прямоугольник `maxW × maxH` с сохранением пропорций (алгоритм `fitBox`).
3. **Адаптивность.** Пределы зависят от ширины колонки переписки и высоты окна
   (реактивно: `winW`/`winH` обновляются на resize; `listWidth` — ширина панели чатов).
4. **Один размер для лоадера и итога.** Превью загрузки (`pendingUploads`) и итоговое
   сообщение используют один `photoBox(aspect)` — без «скачка»/уменьшения при загрузке.
5. **Мгновенная подложка.** До появления картинки — blur-up микропревью (`microBg`,
   ~24px JPEG data-URI из `/uploads-dims`) или средний цвет; спиннер поверх, пока грузится.

## Алгоритм `fitBox(aspect, maxW, maxH)`

```
a = aspect > 0 ? aspect : 1.4
w = maxW; h = w / a
if h > maxH: h = maxH; w = h * a      // portrait упёрся в высоту
if w < 140:  w = 140; h = min(w/a, maxH)  // очень узкое/высокое — минимум по ширине
→ round → { width, height } в px
```

Следствия:
- **широкое** (aspect > maxW/maxH): ширина = `maxW`, высота маленькая → тянется в ширину;
- **квадрат** (aspect ≈ 1): сторона = `min(maxW, maxH)` = `maxH`;
- **высокое** (aspect < 1): высота = `maxH`, ширина = `maxH·a`.

## Пределы (caps)

Ширина колонки: `convWidth = max(240, winW − nav − list)`, где `nav = 72` (левая
нав-панель на десктопе), `list = listWidth` (панель чатов на десктопе), на мобиле оба `0`.

| Тип     | maxW                                 | maxH                          |
|---------|--------------------------------------|-------------------------------|
| Фото    | `clamp(convWidth·0.6, 220, 608)`     | `min(480, winH·0.5)`          |
| Видео   | `clamp(convWidth·0.6, 220, 608)`     | `min(544, winH·0.58)`         |
| Альбом  | ширина сетки = `photoCaps.maxW`      | плитки `aspect-square` (thumb)|

- `0.6` — фото занимает ~60% ширины колонки, но не шире `608px` даже на очень широких
  мониторах (компактнее, чтобы точно влезало в чат).
- Видео чуть выше по высоте (основной контент).
- Мин. ширина `140px` — чтобы очень узкое/высокое фото оставалось кликабельным и под
  подпись.

## Особые случаи

- **Кружки (video notes)** — всегда фиксированный круг `19.5rem × 19.5rem`, в стратегию не
  входят (не меняют размер никогда).
- **Альбомы** (2+ фото) — сетка квадратных плиток (`albumCols`: ≤4 → 2 колонки, иначе 3;
  для 3 фото первое — во всю ширину). Плитки грузят лёгкие thumb (не полноразмер) — их
  много и они маленькие. Ширина сетки = `albumWidth()`.
- **Одиночное фото** — грузит полноразмер (`u`, до 1600px, q85) для резкости; blur-up
  подложка снизу.
- **Лоадер-превью отправки** резервирует ещё и слот аватара в группе (`items-end gap-2` +
  аватар), чтобы при появлении реального сообщения не было горизонтального сдвига.

## Что править при изменениях

- Сделать фото крупнее/мельче → коэффициент `0.74` и потолок `760` в `photoCaps`.
- Видео выше/ниже → `videoCaps` (`0.72`, `680`).
- Ширина альбома → `albumWidth()` (= `photoCaps.maxW`).
- Не забывай: любой новый путь рендера медиа должен брать размер из `photoBox`/`fitBox`,
  а НЕ из процентов/`max-w`, иначе появится «скачок» при загрузке.
